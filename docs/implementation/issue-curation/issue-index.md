# Open issue index for the 4.0 release

Snapshot refreshed 2026-07-16: 103 open issues in `klum-dsl/klum-ast`, inspected from repository base `65f7a99b` and the live issue tracker. The original inventory was read-only; later confirmed normalization actions are recorded below.

All dispositions are provisional maintainer recommendations. “Relevant” means that the current source still lacks the requested capability or still contains the reported risk; it does not mean that the issue is accepted product policy. The historical 2.x/3.x sections of `wiki/Roadmap.md` were not used as current policy.

Compatibility abbreviations:

- `AST`: generated signatures/compiler behavior; verify Groovy 3, 4, and 5. Java 17 is the unchanged baseline.
- `RT`: runtime behavior; verify the affected module on all three Groovy lanes, with no expected Java-version delta.
- `J17/API`: changes Java-visible generated/public signatures and can be source/binary breaking.
- `Gradle`: needs plugin scenario coverage across supported Groovy versions and an explicit Gradle compatibility matrix.
- `IDE`: IDE-version-specific; Java/Groovy runtime lanes do not establish compatibility.
- `Docs`: no runtime compatibility impact.

The evidence key in each row supplies the relevant ADR/wiki/test set; directly related issue numbers are named in the row. A PR is named only where the issue discussion or local merge history establishes a direct relationship (principally PR #429). Where no PR is named, no direct PR was identified in the issue body/comments or local merge history; this is not a claim that no unlinked historical PR ever touched the same code.
